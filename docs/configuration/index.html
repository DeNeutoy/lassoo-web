    Using other encoders#

    By default, lassoo embed uses Scikit Learn's TF-IDF Vectorizer to encode text into sparse vectors. This is very fast and often provides a very useful view of your data, making it very easy to identify near duplicates and text with superficial similarities.

    Lassoo also supports using models from the sentence-transformers library. Sentence Transformers provides a straightforward interface to large neural network models for encoding text into vector representations. You can use any sentence transformer model to encode your data by passing the name of model as the encoder argument to lassoo embed.

    A good default if you want to use a transformer model for embedding is paraphrase-distilroberta-base-v1. It is trained on millions of semantic textual similarity examples and is an effective embedding method for many "general" applications.

    You can see the available pretrained sentence encoder models here. sentence-transformers has transformer based models for over 100 languages and many different use cases.

    Enabling Full Text Search#

    Lassoo also allows full text search over your data as a method for selecting data in the interface. This can be useful if you know certain keywords are relevant for your labels. Combining this with lassoo's recommendation feature can be very powerful. For example, you might search for "TV", which will return all examples containing the phrase. However, because the recommendation uses vector similarity to suggest examples, you can find related examples which don't contain the exact phrase "TV", but do contain relevant terms like "netflix", "movie" etc.

    UMAP Parameters#

    UMAP is a very effective algorithm for unsupervised dimensionality reduction. However, the clustering you achieve from it can vary in how tightly it clusters sub-regions of the space, and how much it preserves global vs local structure. Luckily, UMAP is very fast, so you can try out some different values of the --n_neighbors and --min_dist parameters when using the lassoo embed command (or just pass the same parameter names to your umap.UMAP model, if you are creating it separately) to see which generate the most useful clustering for your data analysis/annotation.

    This blog post by Google PAIR (People and AI Research) is very useful for quickly understanding the meaning of the two most important UMAP parameters.
